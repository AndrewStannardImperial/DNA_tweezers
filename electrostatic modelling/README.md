# Recognition Energy

This folder is a cleaned, GitHub-facing subset of the recognition-energy workflow from the main `Recognition` directory.

## Contents

- `dna_calculator.py`: single calculator module supporting both workflows: fixed `TH` on the instance and `TH` passed into methods.
- `MC_sampling.py`: Monte Carlo sampling pipeline for Mg/Ca coalignment and recognition-energy uncertainty bands.
- `fit_data.py`: fits the Mg/Ca experimental coalignment data and saves `phat` and covariance matrices.
- `load_and_plot.py`: loads the confidence-band CSVs generated by `MC_sampling.py` and plots the best-fit curves with experimental data.
- `forster_radius_fit.py`: fits the experimental Mg/Ca Forster-radius concentration dependence and writes `FR_fit.pdf`.
- `mc_outputs/`: large intermediate Monte Carlo arrays written by `MC_sampling.py`.
- `mc_band_csvs/`: lightweight confidence-band CSVs written by `MC_sampling.py` and read by `load_and_plot.py`.

## Dependencies

The scripts use standard scientific Python packages:

- `numpy`
- `scipy`
- `matplotlib`
- `pandas`
- `tqdm`
- `joblib`

## Typical usage

Generate the Forster-radius fit figure:

```bash
python forster_radius_fit.py
```

Fit the Mg/Ca experimental data and save `phat`/covariance:

```bash
python fit_data.py
```

By default, `fit_data.py` runs differential evolution followed by a local L-BFGS-B polish so the saved fit is less sensitive to the exact differential-evolution stopping point. To save the raw differential-evolution endpoint instead, use:

```bash
python fit_data.py --raw-de
```

Generate the Monte Carlo outputs and confidence-band CSVs:

```bash
python MC_sampling.py
```

`MC_sampling.py` reads `fit_results.npz`, which is created by `fit_data.py`. Run the fitting script first whenever you want the Monte Carlo sampling to use updated best-fit parameters and covariance matrices.

`MC_sampling.py` is parallelized with `joblib`. By default it uses one fewer than the available CPU cores, capped at 60 workers. On a laptop, you may want to use fewer workers so the machine stays responsive:

```bash
MC_N_JOBS=4 python MC_sampling.py
```

On a workstation or cluster node, set `MC_N_JOBS` to the number of CPU cores you want the script to use. Avoid setting it higher than the number of physical/logical cores available, because oversubscribing workers can make the run slower.

Then plot the generated Mg/Ca confidence bands and best-fit theory curves:

```bash
python load_and_plot.py
```

`load_and_plot.py` reads the CSV files generated by `MC_sampling.py`.
